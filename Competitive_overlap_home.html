QSR Market Analysis

Sonic Drive-In vs. Wendy's

Final Technical Report

Project description: This project evaluates the market overlap of two popular quick-service restaurants in Texas: Sonic Drive-In and Wendy's.


ETL Breakdown:

Extraction: We collected each restaurant's address information by web scraping their respective store locations page. In order to do this chromedriver was utitlized to navigate to each restaurant's store location page which contained a list of various links for each city in Texas where a store is located. Chromedriver and Python looping were utilized to navigate to each city's page containing each store address for extraction. To create the web scraping scripts, multiple Python libraries were used including: BeautifulSoup, requests, and datetime. After scraping the data, we exported the Wendy's data set into a csv file, then read in the csv in order to fullfill the project requirements.

Transformation: First, we converted both data sets into a Pandas dataframe. As for cleaning the Sonic data, we did the following: (1) Renamed columns headings, (2) Dropped duplicate addresses, (3) Removed extra commas, and (4) Excluded any non-Texas states. Then, we merged the two separate dataframes with an innerjoin. Next, we utilized a .group_by() method by zipcode. Finally, we used a .count() to calculate the market overlap.

Load: We decided to use MySQL, a relational database, for our production database. This decision was made to take advantage of MySQL's table based database, which stores data in columns and row formats. While there are benefits to non-relational databases, such as cheaper costs and ease of scale, our data fit this option best. In order to load the data the Python libraries SQLAlchemy, PyMySQL, and Pandas were used in the following process. (1) SQLAlchemy was used to create and connect an engine. (2) PyMySQL was used to create our database "etl_project_db". (3) Pandas was used to perform a df.to_sql import of the merged Pandas dataframe to create a populated table in the "etl_project_db" database.
